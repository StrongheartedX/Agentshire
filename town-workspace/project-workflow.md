# 复杂项目 Plan 设计规则

> 本文件只管一件事：如何设计 `create_plan` 的 steps。

## 阶段定义

| # | 阶段 | 必需 | 产出 | 批次 |
|---|------|------|------|------|
| 1 | 方案策划 | ✅ | 设计文档（写入 projectDir/*.md） | batch 1 |
| 2 | 骨架初始化 | ✅ | 项目框架 + 入口文件 + MODULES.md（目录结构 + 模块职责 + 开发分工建议） | batch 2 |
| 3 | UI 设计 | 项目有界面时 | 视觉规范（配色、字体、风格等） | batch 2 |
| 4 | 开发实现 | ✅ | 按骨架的模块定义和分工建议并行实现功能 | batch 3 |
| 5 | 整合联调 | ✅ | 对照设计文档验收 + 修复问题 + 交付可运行成品 | batch 4 |

**上游产出** = 设计文档 + MODULES.md + 视觉规范（若阶段 3 存在）

## 依赖关系

- 方案策划是起点（batch 1），骨架和 UI 设计都依赖它
- 骨架初始化和 UI 设计互不依赖，同属 batch 2，并发执行
- 开发实现（batch 3）必须等 batch 2 全部完成
- 整合联调（batch 4）是最后一步
- UI 设计只在项目有用户界面时需要（游戏/网站/应用 → 需要；CLI/数据处理/后端 → 不需要）

## create_plan 示例（游戏项目）

```json
[
  { "id": "plan",      "batch": 1, "description": "方案策划", "agents": [{ "name": "橙子", "task": "..." }] },
  { "id": "scaffold",  "batch": 2, "description": "骨架初始化", "agents": [{ "name": "岩", "task": "..." }] },
  { "id": "ui",        "batch": 2, "description": "UI设计", "agents": [{ "name": "海棠", "task": "..." }] },
  { "id": "develop",   "batch": 3, "description": "开发实现", "agents": [{ "name": "点点", "task": "..." }, { "name": "小烈", "task": "..." }] },
  { "id": "integrate", "batch": 4, "description": "整合联调", "agents": [{ "name": "岩", "task": "..." }] }
]
```

注意：开发步骤**不需要传 files 字段**——文件边界由架构师在骨架阶段的 MODULES.md 中定义，系统会自动注入给开发和联调阶段。

## 各阶段要点

**骨架初始化**产出要求：
1. 项目工程框架文件（入口 HTML、目录结构、基础模块文件）
2. MODULES.md —— 必须包含：
   - 完整的目录结构树
   - 每个模块的职责说明
   - 开发分工建议 —— 阅读 PROJECT.md 计划概要中开发阶段的人员和专长，按每个开发者的 specialty 划分互不重叠的文件组，格式如：
     - 前端开发负责：js/engine/Renderer.js, js/ui/
     - 后端开发负责：js/systems/, js/entities/

**开发实现**（可多人并行）：
- 按骨架 MODULES.md 的分工建议实现，在骨架已有的目录结构上开发
- 禁止创建 MODULES.md 中未定义的新目录结构

**整合联调**（最终质量保障）：
1. 对照设计文档（SPEC.md / DESIGN.md），逐条核对功能是否实现
2. 打开入口文件，确认页面能正常加载无报错
3. 操作核心功能，确认可交互、无白屏、无控制台报错
4. 若有视觉规范，核对配色、字体、布局是否符合
5. 检查各模块的 import/引用路径是否正确连通
6. 发现问题必须修复，不能只报告不动手——这是最后一道关

## 分工

按 `town-defaults.json` 中的 specialty 匹配居民到对应阶段：

| 阶段 | 匹配 specialty |
|------|---------------|
| 方案策划 | 产品经理 |
| 骨架初始化 | 架构设计 |
| UI 设计 | UI设计 |
| 开发实现 | 前端开发、后端开发 |
| 整合联调 | 架构设计 |

## 禁止

- ❌ 跳过方案策划直接开发
- ❌ 没有任何文件产出
- ❌ 跳过整合联调直接交付
- ❌ 骨架不输出 MODULES.md（开发阶段无法分工）
- ❌ 开发阶段创建骨架未定义的新目录结构
- ❌ 整合联调只报告问题不修复
